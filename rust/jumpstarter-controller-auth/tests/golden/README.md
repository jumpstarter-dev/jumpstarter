# Golden vectors from the Go controller

These fixtures are **generated by the Go reference implementation** and lock
the Rust port of the controller auth stack to the deployed Go behavior. They
are consumed by `tests/golden.rs` and by unit smoke tests in
`src/go_compat/`.

| File | Contents |
|---|---|
| `derivation.json` | ~10 `CONTROLLER_KEY` seeds (empty, ASCII, UTF-8 multibyte, fixed binary) → the ES256 key each derives via `internal/oidc.NewSignerFromSeed` (`sha256(seed)` → seeded Go `math/rand` → `keygen.ECDSALegacy(P256)`). `d_hex`/`x_hex`/`y_hex` are 32-byte left-padded big-endian hex; `sha256_hex`/`seed_int64` are recorded intermediates for stage-pinpointing on failure. |
| `tokens.json` | A Go-signed internal ES256 token (seed `golden-controller-key`, subject `client:default:golden:uid-1234`, issuer/audience defaults from `cmd/main.go`, exp +100y), the exact `GET /jwks` body the Go signer serves, and two router HS256 tokens (`ROUTER_KEY=golden-router-key`, Dial/Listen claim set, exp +100y). |
| `gorand.json` | Raw Go `math/rand` fixtures for the `go_compat::gorand` port: per seed, the first 20 `Int63()` values, the first 64 `Read()` bytes, and the same 64 bytes read in 7+3+54-byte chunks (locks `readVal`/`readPos` persistence across calls). |

## Regeneration

Requires a Go toolchain (same major version as `controller/go.mod`):

```sh
cd controller
go run ./hack/goldenvec -out ../rust/jumpstarter-controller-auth/tests/golden
```

`derivation.json` and `gorand.json` are fully deterministic and must be
byte-identical across regenerations (modulo a Go stdlib change, which would
itself be a finding). `tokens.json` embeds `iat`/`nbf`/`exp` from the
generation time, so its tokens change on every run; the Rust tests only check
those claims structurally (`exp - iat == 100 years`, `nbf == iat`).

## Reverse verification (Rust-minted token checked by Go)

`goldenvec` also has a verification mode for CI jobs that have a Go
toolchain, so Rust-minted internal tokens can be validated by the Go
validator:

```sh
cd controller
go run ./hack/goldenvec \
    -verify-token "$JWT" \
    -seed "$(printf '%s' 'golden-controller-key' | base64)"
# optional: -issuer https://localhost:8085 -audience jumpstarter
```

Exits 0 and prints `token OK` when the token verifies against the key derived
from the seed (issuer + audience + ES256 enforced); exits 1 otherwise.

Do not edit these files by hand.
