# Research: Document Pytest Class Usage

## Existing Pytest Integration

**Decision**: Document the existing `JumpstarterTest` class and pytest fixtures.

**Rationale**: The pytest integration already exists in the codebase but lacks documentation. Need to identify the public API and document it.

**Alternatives considered**: None -- documentation of existing functionality.

## Documentation Structure

**Decision**: Create a single guide page covering installation, basic example, configuration, and local vs distributed modes.

**Rationale**: A single comprehensive guide is easier to find and follow than scattered references across multiple pages.

**Alternatives considered**:
- API reference only: Doesn't help new users get started.
- Multiple pages: Over-fragmenting for the scope of content.
